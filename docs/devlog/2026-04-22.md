### Key Technical Insight
Adding a new `td` project to the pipeline is a **two-place edit**, and both must agree:
1. `PROJECT_MAP` in `td_do_puller.py` (Mac-side тАФ maps project key тЖТ local repo path for `td create`)
2. The `project not in {...}` set in `do-server/server.py` (droplet-side тАФ rejects unknown values at enqueue time)

If only the Mac side is updated, the phone gets `HTTP 400` before anything hits the queue. If only the DO side is updated, the puller will enqueue and then silently skip unknowns as failures. The droplet also isn't a git repo тАФ the canonical copy lives in `td-sync/do-server/`, and deploy is a manual `scp` + `systemctl restart td-capture.service`.

Also: `gaylon.photos` resolves to Cloudflare, not the droplet тАФ SSH must be by IP (`134.199.211.199`), not hostname.
